# 项目唯一执行真相源

以后执行本项目任何任务前，必须先读取本文件，再读取 `TASK.md`、相关 `skills/*/SKILL.md`、`docs/current-status.md` 和 `DEVELOPMENT_LOG.md`。

`AGENTS.md` 是长期通用协作规范；本文件是店小秘自动上架项目的业务执行真相源。若聊天历史、旧日志、旧报告与本文件冲突，以本文件和当前 `TASK.md` 为准。

异常规则统一入口是 `docs/exception-rules.md`。下拉字段、类目安全相邻、native 保存暴露字段、价格来源和 WebBridge / tab-control 中断归类，均按该文件执行。

## 当前固定执行顺序

```text
Amazon 采集前过滤 -> 店小秘采集箱 -> 认领到速卖通海外托管目标店铺 -> 编辑页 -> 类目证据/规则判断 -> 字段填写 -> preflight -> 保存到待发布
```

当前 Recovery / 编辑到待发布阶段禁止执行：

1. 不发布。
2. 不一键发布。
3. 不重新采集已成功认领商品。
4. 不重新认领已成功认领商品。
5. 不处理误认领到 `产品开发 / 草稿箱` 的商品，除非用户明确授权。

## 当前字段硬规则

1. 类目判断必须前置：只有已经成功做到待发布/上架成功并沉淀的同类产品类目，才可以直接复用；其他所有没有成功过的产品，即使有候选规则、历史搜索结果、店小秘候选或 AI 判断，也必须先去速卖通验证品类，再回店小秘编辑页选择可见叶子类目。
2. 价格/货值不是全局固定公式，必须按当前 `TASK.md` / 批次配置声明的价格源、展示价候选策略、汇率、倍率或阶梯倍率、舍入规则计算；Amazon 展示价候选默认取最高有效值，候选包括当前价、区间价、变体价、划线价/List Price；任务未明确价格参数时禁止保存，不能套用历史任务公式。
3. 产品属性只填红色 `*` 必填字段；非红星字段默认不填写、不优化、不作为 preflight 阻塞点。
4. `品牌(Brand Name)` 必填时优先选择 `NONE/None/No Brand` 等无品牌安全值；若页面没有安全可选项，记录字段失败并继续下一个商品。
5. `功能(Function)` / `特性(Feature)` 红星必填且是下拉字段时，有准确项选准确项；没有准确项直接选真实下拉选项 `Other/其他`，不做过度分析；下拉字段的输入框只作为搜索/过滤入口，最终必须选择页面已有选项。
6. `高关注化学品(High-concerned chemical)` 必填时优先选择 `天然未处理(None)` 或页面等价安全值。
7. `产地(Origin)` 必填时，当前 Amazon US -> 速卖通海外托管批次默认选择 `美国(Origin)(US(Origin))` / `United States` 等页面等价真实选项；必须以页面真实读回为准，不允许自动回退 `Mainland China/中国大陆`。
8. `材质(Material)` 没有红色 `*` 时不填写、不点击、不作为失败原因。
9. `材质(Material)` / `框架材质(Frame Material)` 有红色 `*` 时必须填：先提取 Amazon 材质证据；下拉有完全匹配则选完全匹配；无完全匹配则选相近材质；金属网格/钢制收纳优先真实下拉选项 `Metal` / `Steel` / `Iron` / `Alloy` / 等价项；没有相近材质就直接选一个可用材质项；下拉没有选项但字段可输入时，直接输入 Amazon 材质；只有选择和输入都失败时才记录 `material_required_unfillable` 并继续下一个商品。
10. `是否可定制(is_customized)`、`认证(Certification)` 等没有红色 `*` 时不填写、不点击、不作为失败原因。
11. 自定义属性默认清空/删除；不改写 Amazon bullet，不因 `自定义属性值不能超过70个字符` 增加人工优化工作。
12. 产品图片区域必须点击营销图片 `一键生成`，生成 `1:1 白底图` 和 `3:4 场景图`；失败记录 `marketing_image_generation_failed` 后继续按规则处理。
13. 变种参数只勾选红星必填 `发货地`，固定选择 `美国(United States)`；`Color`、`Number of Pcs`、`Size` 非必填时不选。
14. 变种信息中 `销售方式/打包出售` 非必填，默认不勾选、不分析、不作为阻塞点。
15. 商家仓库存默认填写 `15`；旧日志中出现的 `10` 只代表历史结果，不再作为当前执行规则。
16. SKU 编码填写当前 ASIN。
17. 重量和尺寸按采集值或项目默认值填写；缺失时用项目既定兜底规则并记录来源。
18. 运费模板是必填字段，固定选择 `111`；必须通过下拉真实选择并在保存前读回为 `111`。
19. 所有红星必填 Ant 下拉字段必须执行真实选择：关闭旧浮层、打开目标字段下拉、读取真实选项、点击真实选项、读回选中标签；普通文本/数字输入框按字段规则输入，不能把下拉搜索框里的文字当成已选值。
20. Function / Use / Feature 默认先读店小秘推荐可见选项；搜索后出现 `暂无数据` 时必须清空搜索词恢复推荐选项。动态下拉点击真实选项后不得再 Enter/Tab 二次确认；如果读回纯数字/内部 ID，必须清空并重选真实 label。
21. 脚本 preflight 通过不等于 native 保存通过；native `请选择产品属性` 必须定位具体字段并修复。checkbox/radio 组必须按真实选项勾选并读回 checked 状态。
22. 价格/货值只能来自当前任务确认的 Amazon 页面展示价格 USD 与当前任务价格参数；禁止使用店小秘页面旧价格、缓存价格、minPrice/maxPrice、无上下文 UI 数字扫描或手动 CNY 覆盖。

## 失败处理硬规则

1. preflight 字段确实无法自动补齐时，记录字段级失败并继续下一个商品，不等待人工。
2. 店小秘搜索出多个候选类目时，只记录 `dxm_candidate_category_split`，这是中间状态，不允许直接跳过，必须回到速卖通平台和 Amazon 证据裁决。
3. 已完成速卖通相似商品验证后，若速卖通自身类目仍严重分裂且无法判断，才记录 `aliexpress_category_evidence_split` 并继续下一个商品。
4. 已有明确速卖通类目证据或成功沉淀规则后，店小秘仍搜索不到对应可见类目时，才记录 `dxm_visible_category_not_found` 并继续下一个商品；没有速卖通证据时只能记录 `category_evidence_missing` / `needs_aliexpress_category_verification`。
5. 候选规则、历史搜索结果或 AI 判断没有成功待发布/上架验证时，记录 `category_rule_not_success_verified` 或 `aliexpress_category_required`，先去速卖通验证，不直接进入保存。
6. 下拉浮层串场、WebBridge 多标签焦点混乱、页面滚动函数被覆盖等，归类为环境/页面控制异常；不要误判为商品业务失败。
7. 每次任务结束必须执行能力沉淀流程：任务总结、能力抽取、写入/更新 `skills/`、更新 `DEVELOPMENT_LOG.md` 和 `docs/current-status.md`。
8. 保存后跳转到 `/web/smtlocalProduct/offline` 属于成功路径之一；批量恢复优先 direct edit URL，并以 offline / draft 权威列表读回为准，不因 WebBridge 控制异常扩大业务动作。
9. 截图默认是临时证据，不作为长期项目状态。每次任务结束后必须运行 `tools/cleanup-task-screenshots.js plan` 检查未引用截图；未被运行报告、文档或 JSON 明确引用的截图应使用 `cleanup --write` 清理。只有最近失败现场、页面结构变化、关键读回证明等被报告引用的截图可以保留。
10. 每次工作任务执行结束、暂停或异常中断后，必须关闭本次任务自动打开或接管的 Amazon、AliExpress、店小秘编辑页、待发布读回页和临时验证页；下一次执行必须重新打开 direct edit URL 或权威列表页，并重新读回 ASIN、来源 URL、类目、价格和库存后再动作。关闭前必须确认没有未保存的编辑页和未读回的成功结果；如存在未保存字段或保存结果未确认，不得静默关闭，必须记录未关闭原因。

## AliExpress 类目证据通过线

1. `postCategoryId` 共识 `>=80%` 且有 DXM 候选类目，才是高可信通过：`aliexpress_verified` / `high_confidence`。
2. `60%-79%` 只能条件通过：必须有 DXM 候选类目、用途/形态一致、无明显冲突、非风险品类；记录 `conditional_verified` / `low_confidence`。
3. `<60%` 不直接判死，必须进入标题语义二级判断；标题语义一致时优先打开高度相似 AliExpress 详情页补证据，读取规格 `类型` / `类别` / `产品类型` / `商品类型` / `专用工具类型` / `Product Type` / `Category` / `Special Tool Type`。2 条一致/等价详情证据即可通过；若单个详情页缺字段、出现 `浏览/Browse` 导航噪音或被验证页拦截，可以最多补看 5 个同类候选。
4. `<60%` 且 2 个详情页类型一致或等价时，记录 `detail_verified` / `detail_low_confidence` / `detail_type_consensus`；详情页类型可以作为店小秘分类弹窗自动搜索词，但最终保存仍必须经过真实类目回填和 final preflight。
5. `<60%` 但 AliExpress 搜索结果标题语义、用途和形态高度一致时，不能记录为普通 `evidence_split`；详情页类型无法读取且缺安全 DXM 候选类目时记录 `semantic_consensus_needs_dxm_mapping`，有安全 DXM 候选类目且无冲突时记录 `conditional_verified` / `semantic_low_confidence`。
6. 遇到 AliExpress 真人验证、滑块验证、验证码或风控页时，记录 `aliexpress_verification_required`，关闭页面并继续下一个 ASIN；不得自动绕过。
7. 只有类目 ID 分散且标题语义也分散、详情页证据缺失/冲突、用途冲突、无安全 DXM 类目或风险筛查不通过时，才记录 `evidence_split` / `needs_manual_review` / `category_evidence_missing`。
8. 条件通过和详情页通过可以进入后续门禁，但不得和高可信通过混写、混报；最终保存仍必须 final preflight 通过。

## 长期硬性规则

以下规则仍然有效，除非当前 `TASK.md` 明确设置更严格的阶段限制：

1. 测试阶段允许单个产品发布验证；正式运行阶段默认批量发布。
2. 同一批商品全部完成编辑并通过检查后，才允许批量发布。
3. 未完成编辑、检查未通过、状态异常的商品禁止发布。
4. 发布前必须清洗特殊字符，禁止 `"`、`′`、`″`，例如 `4.625"` 必须改为 `4.625 inch`。
5. 删除 Brand Story 模块，保留正常产品图和场景图，过滤 Logo 图和品牌图。
6. 选品阶段发现品牌 Logo、水印、品牌名称印刷在产品主体上，直接跳过。
7. 发布成功不代表类目正确；发布前必须自行校验商品类型与平台类目是否匹配。
8. Sink Strainer 不能归到 Kitchen Faucets；水槽过滤器不能归到厨房龙头。
9. 采集和认领前必须声明当前营业执照组和目标店铺；同一营业执照组内禁止重复采集/认领同一产品，跨营业执照组可以按业务价值复用同一产品。
10. 去重范围是 `businessLicenseGroup + productKey`，不是全局 ASIN；详细规则见 `docs/store-license-dedup-rules.md`。
11. 店小秘认领弹窗的平台渠道固定为 `速卖通海外托管`；不论采集什么链接、上架哪个店铺，都只能勾选该分组下面的店铺，禁止选择 `产品开发`、`草稿箱` 或其他渠道。
12. 编辑页自定义属性默认不填写，除非平台强制、类目强制或发布失败明确提示缺少该字段。
13. 优先保证标题、类目、必填属性、PC 描述、营销图、运费模板、品牌过滤正确。
14. 开发任务收尾必须同步更新与实际行为相关的项目文档、状态文档、测试说明或规则说明；不能只改代码后让 `README`、`docs/current-status.md`、`docs/test-plan.md`、`docs/test-results.md`、`DEVELOPMENT_LOG.md` 或本文件继续描述旧状态。
15. 本项目默认 PR 提交语言、Review 沟通语言和维护说明语言为中文；涉及英文 README、代码标识符、外部平台原文、日志原文或上游协作需要时可以保留英文或追加英文摘要，但中文说明必须作为默认维护真相源。

## 2026-06-24 编辑页保存前预检规则 v1.1.37

1. 编辑页保存前必须先做页面可见字段预检，不允许只依赖 payload 或本地报告判定成功。
2. 产品分类必须在编辑页真实选中；如果页面显示未选择分类，必须根据商品关键词自动选择最接近的可见类目，仍无法选择则禁止保存并记录为类目错误。
3. 运费模板必须通过下拉选项真实选择 `111`，不是向输入框写入文本；保存前必须复核页面显示已选中 `111`。
4. 自定义属性默认清空或跳过，不作为发布前必填项；除非平台强制、类目强制或失败提示明确要求，否则不填写自定义属性。
5. 必填属性允许自动补齐保守默认值：品牌优先 `None/No Brand`，高关注化学品优先 `None/天然未处理`；当前海外托管批次产地固定优先 `United States` / `美国(Origin)(US(Origin))` 并读回。历史记录中的 `Mainland China/中国大陆` 不是当前默认值，不能作为自动回退值。
6. 标题、PC 描述、属性、规格、图片文字和即将提交的平台文本字段必须执行特殊字符清洗，禁止 `"`、`′`、`″`、中文引号和智能引号；尺寸写法统一为 `inch`。
7. 保存前必须确认标题不超过 80 字符、PC 描述不少于 500 英文字符、类目已选、运费模板 111 已选、自定义属性为空。
8. 保存失败时，如果页面提示未选择、不能为空、不能超过字符数或特殊字符错误，允许自动纠偏后重试 1 次；仍失败则记录失败原因并跳过商品。
9. 模板继承必须以页面实际显示为准：类目、运费模板和必填属性必须在保存前被页面确认，不能只记录为“已写入”。
10. 本规则为正式执行规则，适用于后续小批量和批量运行。
## 2026-06-25 正式选品、标题和去重规则

1. 标题禁止保留任何品牌名称、商标、品牌型号；保存前和发布前必须再次检查标题是否仍包含品牌词。
2. 如标题仍包含品牌词、商标词或品牌型号，自动重新生成标题，直到符合规则；仍无法清除时禁止保存和发布。

4. 真正需要过滤的是：同一营业执照组下完全相同的 Amazon 链接、完全相同的 ASIN、完全相同的颜色/尺寸/套餐组合、同一商品被重复采集。

6. 同一品类内必须尽量保证商品多样性；默认跳过当前营业执照组内已经采集、认领、待发布、已发布或已跳过的完全相同商品。
7. 不同营业执照组之间可以复用相同产品；卖得较好或高潜力产品原则上允许跨营业执照组再次上架。
8. 如果营业执照组未知，采集/认领前停止并记录 `license_group_unknown`。

## 2026-06-25 店小秘类目优先规则

1. 类目选择必须以店小秘编辑页实际可见、可保存的类目为准，Amazon 标题或搜索关键词只作为辅助。
2. `Pen Holders` 只能作为类目映射规则，不能作为所有商品或所有 `Organizer` 商品的全局固定类目。
3. 仅当标题、图片、用途判断为桌面文具收纳、笔筒、`Pen Holder`、`Pencil Cup`、`Desk Pen Holder`、`Rotating Pen Organizer`、`Rotating Organizer for pencils / office supplies` 时，优先映射到：办公、文化及教育用品 > 桌上收纳用品 > 笔筒(Pen Holders)。
4. 这类商品优先搜索 `Pen Holders`、`笔筒`、`Pencil Holders`、`Pencil Cup`，不得优先用过泛的 `Organizers` 或错误的线缆收纳类目。
5. 仅出现 `Organizer` 不足以命中笔筒类目；线缆、化妆、浴室、厨房、抽屉、衣柜、文件等收纳必须重新判断实际类目。
6. Amazon 原始品类不能直接等同于店小秘平台类目；店小秘类目必须根据商品实际用途、实物形态、图片内容和标题语义重新判断。
7. 如果 Amazon 类目和店小秘类目不一致，以店小秘平台更贴近商品用途的类目为准；只有类目明显错误才拦截，名称不同但用途匹配可以通过。
8. 店小秘存在明确叶子类目时，必须选择叶子类目；没有高置信叶子类目时禁止保存并记录类目待确认。

## 2026-06-30 速卖通类目证据优先规则

## 2026-06-27 新环境标准执行流程

所有电脑迁移、新环境部署、插件恢复和后续验证必须按以下顺序执行，不得跳级：

```text
Environment Recovery
-> Panel Validation
-> Environment Ready
-> Functional Validation
-> Smoke Test
-> 3x10 Validation
-> Production
```

1. `Environment Recovery` 只恢复 Chrome、Tampermonkey、脚本安装、Node、Git 和项目文件，不做业务动作。
2. `Panel Validation` 只确认 3 个业务面板显示、账号、页面、遮挡和异常状态，不采集、不认领、不编辑、不保存、不发布。
3. `Environment Ready` 必须确认浏览器版本、脚本版本、项目目录、文档、Skill、源码、Node、Git 和只读 Console 状态。
4. `Functional Validation` 只验证插件初始化、Hook、监听、按钮绑定和 Console 稳定性；禁止触发采集、认领、编辑、保存、发布。
5. 只有 `Functional Validation` 全部通过后，才允许进入 `Smoke Test`。
6. 只有 `Smoke Test` 通过后，才允许进入 `3x10 Validation`。
7. 只有 `3x10 Validation` 通过后，才允许进入 `Production`。

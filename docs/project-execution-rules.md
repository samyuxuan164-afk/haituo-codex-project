# 项目正式执行规则

本文件沉淀本次测试形成的正式规则，后续自动化执行、插件开发、流程验证都必须遵守。

异常规则统一入口：`docs/exception-rules.md`。下拉字段、类目安全相邻、native 保存暴露字段、价格来源和 WebBridge / tab-control 中断归类，统一按该文件执行。

## 总目标

本项目目标不是绕过店小秘上架，而是由插件和自动化流程代替人工执行店小秘原有上架流程。

标准链路：

```text
Amazon 商品 -> 速卖通以图搜图/相似商品类目证据 -> 店小秘采集箱 -> 认领 -> 编辑页自动填写 -> 保存 -> 待发布 -> 批次检查 -> 发布
```

## 问题处理三步走原则

所有工作问题、项目问题和优化项都必须遵循三步走：

1. 确认问题：先确认问题是否真实存在、属于业务问题、规则问题、代码问题、页面控制异常还是环境异常。
2. 优化问题：只做最小必要优化，不围绕同一个问题反复试错和扩大修改范围。
3. 确认目标达成：优化后必须回到原工作目标验证是否达成，不能停留在“已修改”。

项目禁止在同一个问题上反复修改很多遍但不验证收口。出现这种情况，按工作失误处理。

## 截图清理规则

1. 截图默认是临时证据，不作为长期项目状态。
2. 下一次继续工作以前，以文档、JSON、代码、异常队列和实时页面读回为准，不以旧截图为准。
3. 每次任务结束后必须运行 `tools/cleanup-task-screenshots.js plan` 生成截图清理计划。
4. 未被运行报告、文档或 JSON 明确引用的截图，应使用 `tools/cleanup-task-screenshots.js cleanup --write` 删除。
5. 只有最近失败现场、页面结构变化、关键读回证明、危险动作排除证明等被报告明确引用的截图可以保留。
6. 删除截图不得删除运行报告、JSON 状态、代码、规则、异常队列或价格/证据库。

## 发布模式

1. 测试阶段允许单个产品发布验证。
2. 正式运行阶段默认批量发布。
3. 只有同一批商品全部完成编辑并通过检查后，才允许批量发布。
4. 未完成编辑的商品禁止发布。

## 发布前字符清洗

发布前必须执行特殊字符清洗。

禁止字符：

1. 英文双引号 `"`。
2. 尺寸符号 `′`。
3. 尺寸符号 `″`。
4. 中文引号、智能引号。

转换规则：

```text
4.625"   -> 4.625 inch
4.625″  -> 4.625 inch
4.625 in. -> 4.625 inch
```

清洗范围包括标题、PC 描述、属性、规格、图片文字、详情文案和所有即将提交到平台的文本字段。

## 图片与品牌过滤

1. 删除 Brand Story 模块。
2. 保留正常产品图和场景图。
3. 过滤 Logo 图、品牌图。
4. 选品阶段禁止带明显 Logo 的产品进入采集流程。
5. 发现品牌 Logo、水印、品牌名称印刷在产品主体上，直接跳过。
6. 主图、拼图、包装盒、说明卡、局部放大图、场景图中只要出现可识别品牌名、Logo、商标符号或品牌包装文字，采集前轻量判断必须改为 `跳过` 或 `待复核`，不得直接判为 `允许采集`。
7. 如果品牌文字只出现在 Amazon 标题但图片完全无品牌露出，可以进入标题清洗流程；如果品牌文字出现在图片或包装上，优先按图片品牌风险处理。
8. 图片 / Logo 复核必须批量化，采集前生成候选图复核页或 review sheet。
9. 复核输出必须归类为 `allow_collect`、`needs_review`、`skip_logo_or_brand_risk`。
10. 单品截图只允许作为异常兜底，必须记录触发原因；不得把单品截图当常规检查流程。
11. OCR / 视觉识别可以作为辅助判断，但最终仍必须按项目规则输出上述三类结果。

## 类目正确性

1. 店小秘和平台发布接口不会自动校验产品类目是否业务正确。
2. 发布成功不代表类目正确。
3. 类目正确性必须由自动化流程在发布前自行校验。
4. 发布前必须检查商品类型与平台类目是否匹配。
5. 如果商品类型与类目明显不一致，禁止发布并记录为类目错误。
6. Sink Strainer 不能归到 Kitchen Faucets。
7. 水槽过滤器不能归到厨房龙头。
8. 类目错误不一定会导致发布失败，但会导致上架质量错误，必须在发布前拦截。
9. 桌面 / 办公场景的 adhesive cable clips / cord holder / cable holder，优先验证到 `办公、文化及教育用品(Office & School Supplies) > 桌上收纳用品(Desk Accessories & Organizer) > 桌面理线器(Cable Organizers)`；不要落到 `线缆工具(Cable Tools)` 或 `导电线胶膏(Conductive Wire Glue Pastes)`。

## 速卖通类目证据优先

1. 新品类的类目主入口是速卖通以图搜图 / 相似商品搜索，不是 DeepSeek，也不是旧关键词规则。
2. DeepSeek 只负责商品理解、属性抽取、标题清洗和风险归纳，不能作为最终类目裁判。
3. 本地 category knowledge 只缓存已经由速卖通相似商品证据或店小秘可见类目验证过的结果。
4. 旧本地关键词规则只能用于诊断和候选解释，不能直接推动自动保存。
5. 没有已验证可见类目规则时，不等待人工，必须先自动进入速卖通相似商品 / 以图搜图类目证据流程。
6. 取得速卖通证据后，回到店小秘编辑页搜索并选择可见叶子类目。
7. 10 个不同品类批量测试必须按“先类目证据、再店小秘填写”的顺序执行。
8. 速卖通证据严重分裂时，记录 `category_evidence_split` 并继续下一个商品。
9. 只有在速卖通证据明确或成功沉淀规则匹配后，店小秘既搜索不到对应可见类目、也不存在安全相邻可见类目时，才记录 `dxm_visible_category_not_found` 并继续下一个商品；没有速卖通证据时记录 `category_evidence_missing` / `needs_aliexpress_category_verification`。
10. preflight 字段确实无法自动补齐时，记录字段级失败并继续下一个商品。
11. 商品图片 / 品牌 / Logo 风险属于 Amazon 采集链接前置过滤；已进入编辑批次的商品不再以该原因等待人工。
12. 成功保存到待发布并回读类目正确后，必须把可复用类目规则沉淀到 `skills/category-resolver/learned_rules.json`。

## 速卖通账号与类目复用

1. 默认使用一个稳定登录的速卖通账号和一个固定 Chrome Profile。
2. 不把多账号轮换作为常规自动化方案；多账号只作为人工备用，不写入业务代码。
3. 可以把速卖通首页、搜索页和店小秘页面固定在 Chrome 书签栏或常用标签中，方便 WebBridge 复用已登录会话。
4. 项目不保存速卖通账号、密码、Cookie 或验证码信息。
5. 已经验证并保存成功的类目，后续同类商品直接调用 `skills/category-resolver/learned_rules.json`，不重复图搜。

## 红星必填下拉字段

1. 红星必填产品属性如果是 Ant/select 下拉框，必须真实选择页面选项；普通输入框可以按字段规则输入，但下拉字段的输入框只作为搜索/过滤入口。
2. 正确流程是：关闭旧下拉浮层 -> 打开目标字段下拉 -> 读取真实选项 -> 点击真实选项对象 -> 读回选中标签。
3. `框架材质(Frame Material)` 按 Amazon 材质证据处理；金属网格/钢制收纳优先真实选项 `Metal` / `Steel` / `Iron` / `Alloy` / 等价项，再用 `Other/其他`。
4. `功能(Function)` 按商品用途处理；收纳/整理/办公类商品优先真实选项 `Storage` / `Organizer` / `Office` / 等价项，再用 `Other/其他`。
5. 下拉搜索框里有字但没有选中标签，或原生保存仍报 `请选择产品属性`，都必须视为未完成。
6. 如果目标值搜索后列表为空，要重新打开目标下拉框并读取未过滤的真实选项；可以选择页面里可见的安全相近/通用选项，但仍必须读回选中标签。
7. 只有新品类、低置信、候选冲突、知识库路径搜索不到且无安全相邻类目、保存/发布前检查报错时，才重新走速卖通图搜 / 相似商品类目验证。
8. Function / Use / Feature 默认先读取推荐可见选项，不默认输入搜索词；出现 `暂无数据` 时必须清空搜索词恢复推荐选项。
9. 动态下拉点击真实选项后不得 Enter/Tab 二次确认；如果读回纯数字/内部 ID，清空字段并用 click-only 方式重选真实选项。
10. `Other/其他` 是 Function / Use / Feature 的合法兜底选项；明显错误选项仍必须拒绝。

## native 保存暴露字段

1. 脚本预检通过不等于店小秘 native 保存通过。
2. native 保存可能暴露额外必填字段，如 Use、Feature、Plastic Type、Theme、Product application scenarios。
3. native 提示 `请选择产品属性` 时，必须定位具体字段并修复，不能继续保存。
4. checkbox/radio 组必须按真实选项勾选并读回 checked 状态。
5. 修复后再保存；同一字段连续 3 次聚焦修复失败，才记录字段级失败。

## 编辑保存后弹窗

1. 商品成功保存到待发布后，如果出现 `产品已移入待发布，请在「待发布」中查看！` 弹窗，直接点击右上角 `X` 关闭。
2. 不点击 `创建新产品继续编辑`。
3. 关闭弹窗后回到采集箱 / 待处理列表继续下一个商品。
4. 该弹窗属于保存成功后的流程提示，不应被记录为业务失败。

## 自定义属性策略

1. 编辑页中的自定义属性默认不填写。
2. 自定义属性属于低优先级字段，不作为发布前必填项。
3. 自定义属性经测试容易触发字符长度、特殊字符等校验错误。
4. 自定义属性不影响保存、不影响待发布、不影响正常发布时，统一跳过。
5. 批量运行阶段默认跳过自定义属性，以减少错误率并提升处理效率。

只有以下情况才允许填写自定义属性：

1. 平台强制要求。
2. 类目强制要求。
3. 发布失败明确提示缺少该字段。

## 核心字段优先级

优先保证以下核心字段正确：

1. 标题。
2. 类目。
3. 必填属性。
4. PC 描述。
5. 营销图。
6. 运费模板。
7. 品牌过滤。

## 批量认领规则

1. 当前批次 ASIN 明确、自动认领关闭、未认领列表能匹配本批时，默认使用受控批量勾选和批量认领。
2. 不得把逐条认领作为常规流程。
3. 单条认领只允许作为异常兜底，并必须记录失败原因。
4. `批量认领` 本身不是危险动作；危险的是跨批次裸点、没有本批 ASIN 约束的认领、误点发布或一键发布。
5. 认领弹窗的平台渠道固定为 `速卖通海外托管`；不论采集什么链接、上架哪个店铺，都必须只勾选该分组下面的店铺。
6. 禁止选择 `产品开发`、`草稿箱` 或其他非 `速卖通海外托管` 分组；弹窗默认选中项一律不能直接信任。
7. 脚本必须按 DOM 顺序只读取 `速卖通海外托管` 标题之后、下一个分组标题之前的店铺复选框。
8. 单店铺任务中，如果 `速卖通海外托管` 分组下只有一个可选店铺，自动勾选并确认，店铺名只记录结果。
9. 只有 `速卖通海外托管` 分组下检测到多个可选店铺时，才按配置目标店铺名匹配。
10. 多店铺场景匹配失败必须拦截并记录 `store_uncertain`。
11. 如果发现当前勾选目标属于 `产品开发`、`草稿箱` 或非 `速卖通海外托管` 分组，必须在点确定前阻断并记录 `store_selection_wrong_target`。
12. 批量认领前必须确认目标店铺属于当前任务声明的营业执照组；如果营业执照组未知，停止认领并记录 `license_group_unknown`。

## 耗时量化规则

1. 不再使用“批处理慢”“超过预期”等模糊描述替代数据。
2. 每个阶段应记录具体耗时：采集耗时、认领耗时、Product Understanding 耗时、类目匹配耗时、页面填写耗时、dry-run 预检耗时、保存耗时。
3. 报告必须包含：目标数量、实际数量、总耗时、单品平均耗时、超时阈值、慢点归类。
4. 没有计时数据时，只能写“耗时未量化”，不能写“超过预期”。
5. 页面控制异常、网络超时或浏览器控制失败必须单独归类，不得作为商品业务失败。

## 浏览器任务收尾规则

1. 每次工作任务执行结束、暂停或异常中断后，必须关闭本次任务自动打开或接管的浏览器页面；下一次执行必须重新打开精确页面，不复用旧编辑页或旧搜索页。
2. 关闭范围包括本次任务的 Amazon 商品页、AliExpress 搜索/证据页、店小秘采集箱页、店小秘编辑页、分类弹窗页、待发布读回页和临时验证页。
3. 不关闭用户原有登录基础页、书签固定页、非本任务页面，或用户明确要求保留的页面。
4. 关闭前必须确认没有未保存的编辑页、未读回的待发布成功结果、正在上传/生成的营销图或正在执行的网络请求。
5. 如果存在未保存字段、保存结果未确认、验证码/登录/页面控制异常等未收口现场，不得静默关闭；必须先记录为未保存现场或环境控制异常，再等待恢复或用户确认。
6. 后续继续执行同一产品时，必须重新打开 direct edit URL 或权威列表页，并重新读回 ASIN、来源 URL、类目、价格和库存后再动作。
7. 批次报告必须记录浏览器收尾结果：关闭了哪些页面、保留了哪些页面、未关闭原因，以及下一次应重新打开的入口。

## 店小秘干净编辑页恢复规则

1. 当目标是继续同一商品编辑验证，必须先打开该商品的 direct edit URL，不得复用旧编辑页或只按店小秘域名匹配标签页。
2. 如果自动化主动刷新、关闭、重开或跳转旧编辑页并触发 `离开此网站？` 弹窗，说明当前目标就是丢弃旧页现场，应在 5 秒内选择 `离开`。
3. `离开` 只允许用于自动化主动丢弃旧页、打开干净页的恢复动作；不得用于保存结果未读回、用户手动编辑未确认、上传/生成任务未收口的页面。
4. 新编辑页打开后 20 秒内必须读回 edit id、ASIN、插件版本和 readonly preflight 函数；任一项不匹配或页面未加载，立即停止当前业务动作并重开干净页。
5. 干净页校验通过前，不允许执行字段编辑、保存并移入待发布、发布、一键发布、重新采集或重新认领。
6. 该流程由 `tools/dxm-live-edit-helper.js forceEdit` 执行；版本号必须按本次任务实际目标传入，不写死在业务规则里。

## 2026-06-25 正式选品、标题和去重规则

1. 标题禁止保留任何品牌名称、商标、品牌型号；保存前和发布前必须再次检查标题是否仍包含品牌词。
2. 如标题仍包含品牌词、商标词或品牌型号，自动重新生成标题，直到符合规则；仍无法清除时禁止保存和发布。
3. 同一款产品不同颜色允许作为不同商品分别上架；不同颜色视为不同 SKU，不属于重复商品，不允许过滤。
4. 真正需要过滤的是：同一营业执照组下完全相同的 Amazon 链接、完全相同的 ASIN、完全相同的颜色/尺寸/套餐组合、同一商品被重复采集。
5. 默认选品任务参数可配置；当前默认值为价格区间 5-20 USD、目标 100 个商品、每个品类最多 15 个商品，但执行时必须以面板/任务配置为准，不得写死。
6. 同一品类内必须尽量保证商品多样性；默认跳过已经采集过的完全相同商品。

## 2026-06-30 营业执照组去重与跨执照复用规则

1. 去重范围是 `businessLicenseGroup + productKey`，不是全局 ASIN。
2. 当前已知结构：一个营业执照组下约 6 个店铺；当前测试只是其中一个店铺。
3. 同一个营业执照组下的任一店铺已经采集、认领、待发布、已发布或已跳过同一 productKey 时，同组其他店铺不得重复采集或重复认领。
4. 不同营业执照组之间可以复用相同产品；卖得较好或高潜力产品原则上允许跨营业执照组再次上架。
5. 跨营业执照组复用必须记录为 `allowed_cross_license_reuse`，并继续经过 Logo/品牌、类目解析、属性、价格、preflight 和保存闸门。
6. 新任务开始时必须声明 `businessLicenseGroup` 和 `targetStore`；如果未声明或无法确认，采集/认领前停止并记录 `license_group_unknown`。
7. 详细知识库以 `docs/store-license-dedup-rules.md` 为准。

## 2026-06-30 Color 变种选择规则

1. Color 不能固定写 `MULTI`。
2. 如果店小秘页面只有一个 Color 选项，选择该真实选项。

6. Color 规则只影响变种 Color 字段，不应改变类目、品牌、高关注化学品、产地、材质、运费模板、自定义属性和保存/发布控制。

## 2026-06-30 价格来源强约束规则

1. 货值 / 供货价只能按当前任务价格参数生成。
2. `Amazon 页面展示价格` 是唯一价格来源，必须来自当前任务输入、Amazon 商品页读取结果或当前任务上下文中明确标记的 Amazon 页面展示价格字段。
3. Amazon 展示价候选包括当前购买价、页面区间价、变体价、划线价 / `List Price`；默认候选策略为 `highest_displayed_value`，即从有效展示价候选中取最高 USD 后再进入当前任务公式。
4. 当前任务价格参数由任务配置提供，包括任务汇率、倍率/阶梯倍率、舍入规则和展示价候选策略；`highest_displayed_value` 不是货值公式，只是默认候选选择策略。
5. 禁止 fallback 到店小秘编辑页价格、无 Amazon 商品页上下文的页面数字扫描、页面缓存价格、采集批次缓存价格、`product.price`、`sourcePrice`、`minPrice`、`maxPrice`。
6. 禁止使用手动 CNY 供货价覆盖作为保存价格来源。
7. 如果当前商品缺少可信 Amazon 页面展示价格，必须阻断预检和保存，记录为价格来源缺失，不能为了继续流程沿用店小秘页面已有价格。
8. 保存前必须校验可见变种货值和 payload SKU 价格等于当前任务价格参数计算结果；不一致时禁止保存。
9. 异常价格必须在发布前单独检查；商品已进入待发布不代表价格自动正确。

## 2026-06-27 新环境标准执行流程

所有电脑迁移、新环境部署、插件恢复和后续验证必须按以下顺序执行，不得跳级：

```text
Environment Recovery
-> Panel Validation
-> Environment Ready
-> Functional Validation
-> Smoke Test
-> 3x10 Validation
-> Production
```

### 阶段边界

1. `Environment Recovery`：只恢复 Chrome、Tampermonkey、脚本安装、Node、Git 和项目文件，不做业务动作。
2. `Panel Validation`：只确认 3 个业务面板显示、账号、页面、遮挡和异常状态，不采集、不认领、不编辑、不保存、不发布。
3. `Environment Ready`：确认浏览器版本、脚本版本、项目目录、文档、Skill、源码、Node、Git 和只读 Console 状态。
4. `Functional Validation`：只验证插件初始化、Hook、页面监听、网络监听、按钮事件绑定和 Console 稳定性；禁止触发采集、认领、编辑、保存、发布。
5. `Smoke Test`：只允许在 `Functional Validation` 全部通过后进入。
6. `3x10 Validation`：只允许在 `Smoke Test` 通过后进入。
7. `Production`：只允许在 `3x10 Validation` 通过后进入。

### 3x10 前置修正项

进入 `3x10 Validation` 前，必须先处理 2026-06-27 的 1 条 Smoke Test 复盘问题：

1. 采集速度偏慢，需要明确可接受耗时或优化执行方式。
2. 采集中频繁调用 Computer Use 效率不足；后续批量验证应尽量减少 Computer Use 调用，只保留边界确认、危险按钮识别和必要人工可视确认。
3. 候选商品主图/包装出现品牌 Logo 或品牌文字时，选品轻量判断必须拦截，不得进入采集。

### 固定禁令

在 `Environment Recovery`、`Panel Validation`、`Environment Ready` 和 `Functional Validation` 阶段，始终禁止：

1. 采集商品。
2. 点击开始采集。
3. 认领商品。
4. 编辑商品。
5. 保存商品。
6. 发布商品。

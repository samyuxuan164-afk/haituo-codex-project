# 项目正式执行规则

本文件沉淀本次测试形成的正式规则，后续自动化执行、插件开发、流程验证都必须遵守。

## 总目标

本项目目标不是绕过店小秘上架，而是由插件和自动化流程代替人工执行店小秘原有上架流程。

标准链路：

```text
Amazon 商品 -> 店小秘采集箱 -> 认领 -> 编辑页自动填写 -> 保存 -> 待发布 -> 批次检查 -> 发布
```

## 发布模式

1. 测试阶段允许单个产品发布验证。
2. 正式运行阶段默认批量发布。
3. 只有同一批商品全部完成编辑并通过检查后，才允许批量发布。
4. 未完成编辑的商品禁止发布。

## 发布前字符清洗

发布前必须执行特殊字符清洗。

禁止字符：

1. 英文双引号 `"`。
2. 尺寸符号 `′`。
3. 尺寸符号 `″`。
4. 中文引号、智能引号。

转换规则：

```text
4.625"   -> 4.625 inch
4.625″  -> 4.625 inch
4.625 in. -> 4.625 inch
```

清洗范围包括标题、PC 描述、属性、规格、图片文字、详情文案和所有即将提交到平台的文本字段。

## 图片与品牌过滤

1. 删除 Brand Story 模块。
2. 保留正常产品图和场景图。
3. 过滤 Logo 图、品牌图。
4. 选品阶段禁止带明显 Logo 的产品进入采集流程。
5. 发现品牌 Logo、水印、品牌名称印刷在产品主体上，直接跳过。
6. 主图、拼图、包装盒、说明卡、局部放大图、场景图中只要出现可识别品牌名、Logo、商标符号或品牌包装文字，采集前轻量判断必须改为 `跳过` 或 `待复核`，不得直接判为 `允许采集`。
7. 如果品牌文字只出现在 Amazon 标题但图片完全无品牌露出，可以进入标题清洗流程；如果品牌文字出现在图片或包装上，优先按图片品牌风险处理。

## 类目正确性

1. 店小秘和平台发布接口不会自动校验产品类目是否业务正确。
2. 发布成功不代表类目正确。
3. 类目正确性必须由自动化流程在发布前自行校验。
4. 发布前必须检查商品类型与平台类目是否匹配。
5. 如果商品类型与类目明显不一致，禁止发布并记录为类目错误。
6. Sink Strainer 不能归到 Kitchen Faucets。
7. 水槽过滤器不能归到厨房龙头。
8. 类目错误不一定会导致发布失败，但会导致上架质量错误，必须在发布前拦截。

## 自定义属性策略

1. 编辑页中的自定义属性默认不填写。
2. 自定义属性属于低优先级字段，不作为发布前必填项。
3. 自定义属性经测试容易触发字符长度、特殊字符等校验错误。
4. 自定义属性不影响保存、不影响待发布、不影响正常发布时，统一跳过。
5. 批量运行阶段默认跳过自定义属性，以减少错误率并提升处理效率。

只有以下情况才允许填写自定义属性：

1. 平台强制要求。
2. 类目强制要求。
3. 发布失败明确提示缺少该字段。

## 核心字段优先级

优先保证以下核心字段正确：

1. 标题。
2. 类目。
3. 必填属性。
4. PC 描述。
5. 营销图。
6. 运费模板。
7. 品牌过滤。

## 2026-06-25 正式选品、标题和去重规则

1. 标题禁止保留任何品牌名称、商标、品牌型号；保存前和发布前必须再次检查标题是否仍包含品牌词。
2. 如标题仍包含品牌词、商标词或品牌型号，自动重新生成标题，直到符合规则；仍无法清除时禁止保存和发布。
3. 同一款产品不同颜色允许作为不同商品分别上架；不同颜色视为不同 SKU，不属于重复商品，不允许过滤。
4. 真正需要过滤的是：完全相同的 Amazon 链接、完全相同的 ASIN、完全相同的颜色/尺寸/套餐组合、同一商品被重复采集。
5. 默认选品任务参数可配置；当前默认值为价格区间 5-20 USD、目标 100 个商品、每个品类最多 15 个商品，但执行时必须以面板/任务配置为准，不得写死。
6. 同一品类内必须尽量保证商品多样性；默认跳过已经采集过的完全相同商品。

## 2026-06-25 店小秘类目优先规则

1. 类目选择必须以店小秘编辑页实际可见、可保存的类目为准，Amazon 标题或搜索关键词只作为辅助。
2. `Pen Holders` 只能作为类目映射规则，不能作为所有商品或所有 `Organizer` 商品的全局固定类目。
3. 仅当标题、图片、用途判断为桌面文具收纳、笔筒、`Pen Holder`、`Pencil Cup`、`Desk Pen Holder`、`Rotating Pen Organizer`、`Rotating Organizer for pencils / office supplies` 时，优先映射到：办公、文化及教育用品 > 桌上收纳用品 > 笔筒(Pen Holders)。
4. 这类商品优先搜索 `Pen Holders`、`笔筒`、`Pencil Holders`、`Pencil Cup`，不得优先用过泛的 `Organizers` 或错误的线缆收纳类目。
5. 仅出现 `Organizer` 不足以命中笔筒类目；线缆、化妆、浴室、厨房、抽屉、衣柜、文件等收纳必须重新判断实际类目。
6. Amazon 原始品类不能直接等同于店小秘平台类目；店小秘类目必须根据商品实际用途、实物形态、图片内容和标题语义重新判断。
7. 如果 Amazon 类目和店小秘类目不一致，以店小秘平台更贴近商品用途的类目为准；只有类目明显错误才拦截，名称不同但用途匹配可以通过。
8. 店小秘存在明确叶子类目时，必须选择叶子类目；没有高置信叶子类目时禁止保存并记录类目待确认。

## 2026-06-28 同品类类目/属性验证修正规则

1. 单个商品只能证明链路能跑通，不能证明类目和属性规则正确。
2. 同品类验证必须使用多个差异化商品，并同时检查：店小秘完整中文父路径、英文叶子类目、必填属性、变种参数、运费模板。
3. `Drawer Organizers` 这类广义收纳商品不能只看英文叶子名；如果完整路径落到“衣物收纳 / 内衣收纳盒”，但商品主图、标题、Amazon 详情没有内衣、衣物、袜子、衣柜等证据，必须降权或拦截。
4. 小型透明抽屉收纳盒、化妆品收纳、办公用品收纳、厨房/浴室杂物收纳，应优先寻找更广义的家居收纳、储物盒、收纳托盘、桌面/办公收纳等路径；只有商品证据明确指向内衣或衣物时，才允许进入内衣/衣物收纳类目。
5. DeepSeek Product Understanding 只能提供商品理解和候选建议；最终类目必须由本地 Rule Engine + 店小秘可见路径共同确认。
6. 材质是必填属性时不得留空。优先从 Amazon 标题、详情、规格信息识别 acrylic、plastic、silicone、rubber、metal、wood、fiberglass 等；缺少明确证据且为透明收纳用品时，允许按项目规则兜底为 Plastic。
7. 当前 Amazon US 到速卖通海外托管流程中，SKU 变种发货地固定为 `United States`，保存前必须补齐。

## 2026-06-27 新环境标准执行流程

所有电脑迁移、新环境部署、插件恢复和后续验证必须按以下顺序执行，不得跳级：

```text
Environment Recovery
-> Panel Validation
-> Environment Ready
-> Functional Validation
-> Smoke Test
-> 3x10 Validation
-> Production
```

### 阶段边界

1. `Environment Recovery`：只恢复 Chrome、Tampermonkey、脚本安装、Node、Git 和项目文件，不做业务动作。
2. `Panel Validation`：只确认 3 个业务面板显示、账号、页面、遮挡和异常状态，不采集、不认领、不编辑、不保存、不发布。
3. `Environment Ready`：确认浏览器版本、脚本版本、项目目录、文档、Skill、源码、Node、Git 和只读 Console 状态。
4. `Functional Validation`：只验证插件初始化、Hook、页面监听、网络监听、按钮事件绑定和 Console 稳定性；禁止触发采集、认领、编辑、保存、发布。
5. `Smoke Test`：只允许在 `Functional Validation` 全部通过后进入。
6. `3x10 Validation`：只允许在 `Smoke Test` 通过后进入。
7. `Production`：只允许在 `3x10 Validation` 通过后进入。

### 3x10 前置修正项

进入 `3x10 Validation` 前，必须先处理 2026-06-27 的 1 条 Smoke Test 复盘问题：

1. 采集速度偏慢，需要明确可接受耗时或优化执行方式。
2. 采集中频繁调用 Computer Use 效率不足；后续批量验证应尽量减少 Computer Use 调用，只保留边界确认、危险按钮识别和必要人工可视确认。
3. 候选商品主图/包装出现品牌 Logo 或品牌文字时，选品轻量判断必须拦截，不得进入采集。

### 固定禁令

在 `Environment Recovery`、`Panel Validation`、`Environment Ready` 和 `Functional Validation` 阶段，始终禁止：

1. 采集商品。
2. 点击开始采集。
3. 认领商品。
4. 编辑商品。
5. 保存商品。
6. 发布商品。

# 100品类100产品压测执行计划与风险边界

当前阶段：Validation / 100品类100产品压测准备。

状态：业务参数已确认；等待用户明确启动后，才允许开始真实店小秘业务动作。

## 已确认压测参数

| 参数 | 确认值 |
|---|---|
| 任务名称 | 100品类压测 |
| 平台 | 速卖通海外托管 |
| 目标店铺 | Halo Home Store |
| 营业执照组 | A1 |
| 目标 | 编辑到待发布，不发布，不一键发布 |
| 产品来源 | Amazon 搜索 |
| 新采集 / 认领 | 允许新采集，允许认领 |
| 价格范围与公式 | Amazon 页面展示价格 USD 5-20；货值 = 页面展示价格 x 7 x 1.55；区间价取最高值 |
| 样本要求 | 100 条，每个类目 1 条 |
| 类目证据 | 需要 AliExpress 类目证据；允许安全相邻类目 |
| 预判分流 | 只对 `auto_ready` 产品执行 |
| 异常处理 | 失败进入异常队列；不反复卡死；字段级记录失败原因 |
| 保存读回 | 保存后必须在待发布列表读回确认 |

## 目标

验证当前自动化链路能否在 100 个不同品类、每个品类 1 个产品的压力样本中稳定完成：

```text
Amazon 候选过滤
-> 类目证据
-> 店小秘采集箱
-> 认领到速卖通海外托管目标店铺
-> 编辑页字段填写
-> preflight
-> 保存到待发布
-> 权威列表读回
```

本轮压测目标不是发布商品，而是验证跨品类采集、认领、类目证据、字段填写、保存到待发布和异常归类能力。

## 当前禁止

在用户明确确认开始压测前，禁止：

1. 打开或操作店小秘业务页面。
2. 重新采集。
3. 重新认领。
4. 编辑或保存商品。
5. 发布、一键发布、采集并一键发布。
6. 处理 `产品开发 / 草稿箱`。
7. 修改业务脚本。

压测开始后仍禁止：

1. 发布。
2. 一键发布。
3. 采集并一键发布。
4. 认领到非 `速卖通海外托管` 分组。
5. 自动回退 `Origin` 到 `Mainland China/中国大陆`。

## 进入条件

开始真实压测前必须确认：

1. 用户明确授权开始 100 品类 100 产品压测。
2. 当前 `businessLicenseGroup` 已声明。
3. 当前 `targetStore` 已声明，并确认属于 `速卖通海外托管`。
4. 当前脚本版本已在页面读回：
   - `DXM Automation V1 - NEW v2.1.2` 或后续明确版本；
   - `DXM Amazon Crawlbox V1 v0.1.29` 或后续明确版本；
   - `save Payload V3 0.6.3` 或后续明确版本。
5. `docs/exception-rules.md` 已作为异常规则入口。
6. 价格公式已由任务配置确认；默认不得从页面旧价格或缓存价格取值。
7. Amazon US -> 速卖通海外托管批次的 `Origin` 默认 United States，不允许自动回退 Mainland China。

如果 `businessLicenseGroup` 或 `targetStore` 无法确认，停止并记录 `license_group_unknown` / `target_store_unknown`。

## 样本设计

1. 目标数量：100 个产品。
2. 品类策略：100 个不同商品族 / 不同品类，默认每个品类 1 个产品。
3. 重复限制：同一营业执照组内不得重复采集/认领相同 `productKey`。
4. 候选过滤：
   - 跳过主图、包装、说明卡、局部图中有明显品牌 / Logo / 商标的产品；
   - 跳过品牌型号不可清洗或疑似侵权产品；
   - 跳过食品、医疗、儿童、带电/电池、侵权高风险产品；
   - 跳过 Amazon 页面展示价格缺失产品；
   - 跳过无清晰主图产品；
   - 跳过变体过复杂产品；
   - 跳过不适合当前海外托管流程的高风险商品。
5. 价格要求：每个产品必须保留可信 Amazon 页面展示价格 USD，且页面展示价格范围必须为 USD 5-20；如果 Amazon 展示价格区间，取最高值判断范围并计算货值。

## 分阶段执行计划

### 阶段 0：只读准备

目标：确认环境可读、规则可用、账号和面板状态正常。

动作：

1. 只读检查脚本版本、面板存在、目标页面状态。
2. 只读确认采集箱、待发布、发布中、发布失败数量。
3. 只读确认 native 自动认领关闭。
4. 生成压测批次 ID，例如 `validation-100-YYYYMMDD-HHMM`。

停止条件：

1. 面板不显示。
2. 脚本版本不符。
3. 自动认领开启且无法关闭。
4. 页面存在危险动作误触风险。

### 阶段 1：候选与类目证据准备

目标：形成 100 个候选产品和对应类目证据，不做店小秘采集动作。

动作：

1. 为每个品类选 1 个 Amazon 候选。
2. 记录 ASIN、Amazon 页面展示价格 USD、标题、图片风险、商品族。
3. 对新品类走 AliExpress / 相似商品类目证据。
4. 对已成功沉淀品类调用 `skills/category-resolver/learned_rules.json`。
5. 输出候选清单和证据状态。
6. 将候选分流为 `auto_ready`、`needs_review` 或具体 `skip_*` 状态；只有 `auto_ready` 进入后续采集。

每个候选至少记录：

```text
asin
productFamily
amazonDisplayedPriceUsd
logoRisk
categoryEvidenceStatus
precheckRoutingStatus
dxmSearchTerms
expectedOrigin=United States
expectedFreightTemplate=111
```

停止条件：

1. 类目证据缺失且无法自动获得。
2. 类目证据严重分裂。
3. 图片 / Logo 风险无法判断。
4. 价格来源缺失。
5. 预判分流不是 `auto_ready`。

### 阶段 2：受控采集

目标：只采集通过候选过滤和类目证据准备的产品。

动作：

1. 分批采集，建议每批 10 个产品。
2. 每批采集前确认输入链接数量和目标数量一致。
3. 只点击明确 `开始采集`，不点击 `采集并一键发布`。
4. 每批结束读回采集成功 / 失败数量。

停止条件：

1. 采集数量少于输入数量且原因不明。
2. native 自动认领被打开。
3. WebBridge / tab-control 控制异常连续出现。
4. 出现发布或一键发布相关确认框。

### 阶段 3：受控认领

目标：只认领当前批次 ASIN 到指定 `速卖通海外托管` 目标店铺。

动作：

1. 按当前批次 ASIN 约束勾选。
2. 认领弹窗只读取 `速卖通海外托管` 分组下店铺。
3. 如果该分组下只有一个可选店铺，自动选择；多店铺时按配置目标店铺匹配。
4. 认领后读回当前批次进入待处理/采集箱编辑队列的数量。

停止条件：

1. 目标店铺不在 `速卖通海外托管` 分组。
2. 弹窗默认选中 `产品开发 / 草稿箱` 或其他错误分组。
3. 多店铺匹配失败，记录 `store_uncertain`。
4. 营业执照组未知。

### 阶段 4：编辑页填写与保存到待发布

目标：逐个产品完成编辑页字段填写、preflight、保存到待发布。

动作：

1. 优先使用 direct edit URL 打开当前产品，避免旧 tab 匹配。
2. 类目必须先有 AliExpress / 相似商品 / 已验证规则证据。
3. 没有精确 DXM 类目时，允许安全相邻类目，记录 `safe_adjacent_dxm_category_selected`。
4. 必填下拉字段必须真实选择选项并读回 label。
5. native save 暴露额外字段时，按 `docs/exception-rules.md` 修复。
6. 货值按当前任务 Amazon 页面展示价格 USD × 汇率 × 倍率计算；区间价取最高值。
7. `Origin` 固定 United States 等价真实选项，不自动回退 Mainland China。
8. 保存只允许点击 `保存并移入待发布`。
9. 保存后用 `/web/smtlocalProduct/offline` 权威读回 ASIN / SKU / 类目 / 价格 / 库存。

停止条件：

1. preflight 未通过。
2. native `请选择产品属性` 未定位或未修复。
3. visible/SKU 价格与期望值不一致。
4. Origin 无法选择 United States 且需要人工规则确认。
5. 类目明显错误。
6. 同一字段连续 3 次聚焦修复失败。
7. WebBridge 控制异常导致无法确认当前 tab / 当前 ASIN。

### 阶段 5：批次读回与报告

目标：只验证是否进入待发布，不发布。

动作：

1. 每 10 个产品生成一个小批次报告。
2. 每 50 个产品生成一次中间汇总。
3. 100 个产品结束后生成总报告。
4. 报告必须区分业务失败、字段失败、类目失败、价格失败、环境控制异常。

每条记录至少包含：

```text
asin
productFamily
status
categoryPath
categoryEvidenceStatus
safeAdjacentUsed
requiredFieldFailures
nativeSaveExtraFields
expectedCnyPrice
visibleCnyPrice
originReadback
freightTemplateReadback
stockReadback
skuReadback
environmentExceptions
finalReadbackSource
```

## 验收标准

压测通过标准：

1. 100 个产品均有完整执行记录。
2. 每个产品的最终状态来自权威列表读回。
3. 保存到待发布的产品，SKU 必须等于 ASIN。
4. 保存到待发布的产品，货值必须等于当前任务公式结果。
5. 保存到待发布的产品，`Origin` 必须是 United States 等价真实读回。
6. 保存到待发布的产品，运费模板必须读回 `111`。
7. 类目安全相邻必须有证据和记录。
8. 环境控制异常不得记为商品业务失败。
9. 未发布、未一键发布、未处理产品开发草稿箱。

建议分级目标：

```text
最低可接受：80/100 保存到待发布，且所有失败均有可复用原因
目标通过：90/100 保存到待发布，0 个价格错误，0 个错误店铺认领，0 个发布动作
理想通过：95/100 保存到待发布，剩余失败均为明确字段/类目/环境异常
```

## 风险边界

### 高风险

1. 误点发布 / 一键发布。
2. 认领到 `产品开发 / 草稿箱`。
3. Origin 自动回退 Mainland China。
4. 使用店小秘旧价格或缓存价格。
5. 类目无证据但强行保存。
6. WebBridge 旧 tab 匹配导致编辑错商品。

处理方式：立即停止批次，记录现场，不扩大业务动作。

### 中风险

1. native 保存暴露额外字段。
2. 下拉字段出现 `暂无数据` 或内部 ID。
3. 安全相邻类目需要人工复核边界。
4. 商品图片品牌风险不确定。
5. 单批采集数量与输入数量不一致。

处理方式：按异常规则库修复或跳过，并进入报告。

### 低风险

1. 单个字段无合适 exact option 但 `Other/其他` 可提交。
2. 类目名称不完全一致但同用途同形态。
3. WebBridge 单次截图/DOM 读取超时但权威列表可读回。

处理方式：继续执行并记录。

## 执行节奏建议

1. 第 1 批：10 个不同品类，小批验证。
2. 第 2-5 批：每批 10 个，累计 50 个。
3. 中场复核：检查失败归类、价格、Origin、店铺认领、类目相邻记录。
4. 第 6-10 批：每批 10 个，累计 100 个。
5. 100 个完成后只生成验证报告，不发布。

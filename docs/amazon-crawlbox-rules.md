# Amazon 采集到店小秘采集箱规则沉淀

## 目标

把亚马逊选品采集从“人工记规则”改成“脚本先过滤、人工只复核低置信内容”的流程。

当前脚本负责先筛掉明显不合规、不重复、不适合继续采集的商品；人工主要确认主图、包装、产品本体是否带 Logo / 商标。

## 本次已沉淀的规则

### 可调整任务规则

这些条件不写死，每次任务可以在插件面板里调整：

- 价格范围
- 最低评分
- 最低评论数
- 最高运费
- 最大发货天数
- 最大页数
- 目标采集数量
- 同类产品数量上限
- 敏感词列表

默认值：

```text
价格：5-20 USD
最低评分：4.2
最低评论数：20
最高运费：0
最大发货天数：10
最大页数：2
同类上限：20
Logo 确认：开启
```

源码要求：`DXM Amazon Crawlbox V1` 默认 `requireLogoApproval` 必须为 `1`。如果浏览器脚本或面板配置显示为 `0`，不得进入 3x10；必须先恢复为开启。

### 必须剔除的商品

脚本会根据标题和页面文字先过滤以下风险：

- 服装
- 母婴
- 食品
- 液体类
- 膏状、霜状、胶状、香皂、固体皂类
- 美妆、护肤、化妆品、指甲贴
- 宠物食品、宠物药物、宠物洗护
- 大 IP、动漫、影视、游戏角色
- 玻璃、易碎品
- 强认证电器
- 化学品、肥料、种子、燃料
- 刀具、危险工具
- 医疗、药品、保健功效类
- 成人敏感类
- 标题或页面信息中明显带 logo、trademark、branded、officially licensed 的商品

### 主图 Logo 规则

亚马逊页面无法只靠文字 100% 判断图片里的 Logo，所以脚本现在采用更安全的策略：

1. 标题首词疑似品牌词的商品默认排除，减少带品牌包装或 Logo 的商品进入后续流程。
2. 扫描后剩余商品默认进入“待 Logo 复核”。
3. 点击“下载Logo复核页”，一次查看所有待复核商品的主图、ASIN、标题和疑似品牌提示。
4. 确认无 Logo / 商标后，把 ASIN 填入“无 Logo ASIN”框。
5. 再扫描一次，只有通过复核的 ASIN 才进入“可采集”。
6. 主图、包装图、拼图局部、说明卡、品牌水印、产品本体或场景图中出现可识别品牌名、Logo、商标符号或品牌包装文字时，采集前结果只能是“跳过”或“待复核”，不能直接判为“允许采集”。
7. Logo / 品牌复核必须批量化：采集前生成候选图复核页 / review sheet，一次查看本批候选。
8. 复核页必须输出分类：`allow_collect` / `needs_review` / `skip_logo_or_brand_risk`。
9. OCR / 视觉识别可以作为辅助判断 Logo、品牌包装、水印、商标文字；识别结果只能作为辅助证据，最终仍以复核分类写入批次。
10. 单品截图只允许作为异常兜底，并必须记录触发原因。

这样可以避免把明显带品牌包装、品牌主图、商标露出的商品推进店小秘。

### 2026-06-27 Smoke Test 漏判案例

ASIN `B0F5WNXRY9` 已成功采集到店小秘采集箱，但复盘发现主图 / 包装图存在 Logo 风险。

处置：

- 记录为：采集成功，但选品过滤漏判。
- 原因：主图 / 包装图存在 Logo 风险。
- 停止该商品后续认领、编辑、保存、发布。
- 后续同类情况必须在采集前拦截为“跳过”或“待复核”。

### 去重规则

去重范围必须按营业执照组判断，不能简单做全局 ASIN 去重。详细规则见：

- `docs/store-license-dedup-rules.md`

脚本记录每个 ASIN / productKey 在当前营业执照组下的状态：

- 已采集
- 已认领
- 已发布
- 已跳过

后续扫描到相同 ASIN 时：

1. 如果同一个营业执照组下任一店铺已经处理过该产品，当前营业执照组内不得重复采集或重复认领。
2. 如果是不同营业执照组，允许再次采集和上架，尤其是卖得较好或高潜力产品。
3. 跨营业执照复用必须记录为 `allowed_cross_license_reuse`，不能误判为同组重复。
4. 新任务开始时必须明确当前营业执照组和目标店铺；如果营业执照组未知，采集前必须停止确认。

### 同类上限规则

同一个类目词默认最多保留 `20` 条已采集 / 已认领产品。

达到上限后，脚本会提示换类目继续补数量，避免同一个营业执照或同类产品重复铺货过多。

## 标准操作流程

1. 在 Amazon 搜索目标类目词。
2. 设置本次任务价格段、评分、评论数、运费、数量等条件。
3. 正式测速点击“快速采集并进店小秘”；只采链接池时点击“快速批量采集当前类目”，单页调试时才点击“扫描当前页”。
4. 查看主图，确认无 Logo 的商品，把 ASIN 填入“无 Logo ASIN”。
5. 再点“扫描当前页”，生成可采集链接。
6. 到店小秘链接采集页，点击“一键采集+认领”。如果页面已有本批 ASIN 对应的未认领商品，可点击“认领当前批次未认领”；不能跨品类直接认领全部未认领。
   - 如果插件已有 Amazon 候选批次，脚本会先自动填入候选链接。
   - 如果店小秘输入框里已经有 Amazon 链接，脚本会直接读取输入框链接并生成本批任务。
7. 脚本自动点击“开始采集”、等待完成并关闭采集弹窗。
8. 脚本自动执行本批受控批量认领：先按本批 ASIN 勾选未认领行，再点击“批量认领”，在弹窗顶部保持 `全部`，到左侧 `速卖通海外托管` 分组下勾选该分组内具体店铺，再读回 `已选(1)` 并点击“确定”。
   - 当前批次 ASIN 明确、自动认领关闭、未认领列表匹配本批时，默认批量勾选并批量认领。
   - 不要逐条认领；单条认领只作为异常兜底，必须记录失败原因。
   - 认领弹窗顶部应保持 `全部`，不要点击顶部 `速卖通海外托管` 标签，也不要先搜索店铺名；顶部标签只是筛选视图，主路径是在 `全部` 页按分组勾选。
   - 认领目标固定为左侧 `速卖通海外托管` 分组下的具体店铺；店铺名不作为主阻断条件，不论店铺显示名是什么，都不能选择 `产品开发`、`草稿箱` 或其他渠道。
   - 脚本不得信任弹窗默认选中项；必须从 `速卖通海外托管` 分组容器里读取具体店铺复选框，不能点该分组的 `全选` 当作替代。
   - 如果该分组下只有一个可选店铺，直接自动勾选该具体店铺并读回 `已选(1)`，店铺名只记录结果。
   - 如果该分组下有多个具体店铺，必须阻断并记录 `store_uncertain`，除非后续业务明确允许多店铺同选。
   - 多店铺匹配失败必须拦截并记录 `store_uncertain`。
   - 如果发现当前勾选目标属于 `产品开发`、`草稿箱` 或非 `速卖通海外托管` 分组，必须在点确定前阻断并记录 `store_selection_wrong_target`。
   - `批量认领` 不是危险动作；危险的是跨批次裸点、没有本批 ASIN 约束的认领、误点发布或一键发布。
9. 认领完成后脚本自动标记已采集 / 已认领，并打开 `产品 > 速卖通海外托管店采集箱` 核对。
10. 如果采集进度超过 30 秒不更新，脚本进入刷新检查状态；刷新后按采集箱结果决定继续、补齐缺口或重新开始。
11. 如果只有个别产品进入采集箱，但这些产品还没有认领成功，可以保留在采集箱，不删除，继续补齐缺口。
12. 如果没有产品进入采集箱，直接重新点击“一键采集+认领”或换下一批链接。
13. 如果批量勾选失败，优先修复批量选择逻辑；临时验证时才允许逐条认领，并必须记录失败原因。
14. 如果数量不足，关闭弹窗，换下一个类目或下一批链接补齐。

## 2026-06-28 WebBridge 机械动作修正规则

1. 店小秘链接采集页的采集输入框是固定流程入口，脚本必须优先识别“网址/链接/产品的网址”相关 textarea，不能靠最大输入框盲猜。
2. 点击“开始采集”前必须校验采集框内 Amazon 链接数量，实际读取数量必须等于本批候选数量；不一致时停止，避免只采到 1 条。
3. 页面存在“采集并一键发布、采集并自动认领、批量认领、一键发布、发布”等按钮，只能记录 warning；只有即将点击危险按钮时才 block。
4. “开始采集”是明确安全目标按钮，可以继续。
5. “批量认领”仅允许在“本批 ASIN 已锁定、已勾选当前批次行、跳过品牌/Logo 风险行”的受控流程中点击；裸点批量认领仍然禁止。
6. 10 条同品类验证应优先使用受控批量认领，不应逐条认领；逐条认领只作为批量选择失败后的临时兜底并必须记录原因。

## 高效流水线能力

从 `v0.1.17` 开始，Amazon 采集插件沉淀为可复用流水线，而不是单次手动流程：

- 点击“快速采集并进店小秘”时，脚本自动记录端到端运行指标。
- 运行指标包括：目标数量、Amazon 批量采集耗时、进入店小秘时间、店小秘采集完成时间、认领完成时间、最终结果、缺口数量。
- 插件摘要区会显示当前耗时，目标时间固定按 `120 秒` 对照。
- 点击“下载运行指标”可以导出本次测速结果，不需要截图判断。
- 运行指标必须量化：目标数量、实际数量、总耗时、单品平均耗时、超时阈值、慢点归类。
- 阶段耗时必须拆分记录：采集耗时、认领耗时、Product Understanding 耗时、类目匹配耗时、页面填写耗时、dry-run 预检耗时、保存耗时。
- 如果某阶段当前工具未记录，报告只能写“耗时未量化”，不能写“超过预期”。
- 如果 Amazon 当前类目没有候选商品，脚本优先打开“类目队列”里的下一个类目；没有队列时才新开当前类目页面。
- 如果店小秘采集只成功一部分，脚本记录 `partial_collection_needs_supplement`，保留成功产品并输出剩余缺口。
- 缺口补齐原则：相同品类未认领成功的产品保留，不删除；继续补足剩余数量即可。

从 `v0.1.18` 开始继续补齐以下自动化规则：

- 店小秘出现“重复采集”弹窗时，脚本自动点“跳过”，不重复采集旧商品。
- 店小秘出现“采集任务完成，成功 0 条”时，脚本自动关闭弹窗，并打开类目队列里的下一个 Amazon 页面补货。
- 采集后如果店小秘行数据暴露 `Brand Name`、`Manufacturer`、`Trademark` 或 Logo/品牌文本，脚本不会勾选认领该行，直接标记为排除并补下一个产品。
- Amazon 搜索入口不再依赖每次手动打开首页，脚本内置默认类目队列；正式任务可直接替换队列内容。
- 队列支持关键词和 Amazon 搜索 URL。关键词会自动套用当前价格范围，例如 5-20 美金。

从 `v0.1.19` 开始，采集插件会把当前批次商品的 ASIN、价格、采集关键词、重量、尺寸写入共享批次数据。共享批次价格只允许作为采集记录和诊断信息，不允许作为主插件上架货值来源。

主插件 `v1.1.16` 的编辑页字段规则：

- 产品分类：优先使用亚马逊采集关键词和商品标题调用店小秘类目搜索接口；未高置信命中时阻断 dry-run，不允许保存/发布。
- 货值：只能按当前任务中的 `Amazon 页面展示价格 × 任务汇率 × 任务倍率/阶梯倍率` 以及当前任务舍入规则、区间价策略计算；Amazon 页面展示区间价时必须读取当前任务 `rangePolicy`，本次 100 品类任务才使用 `highest_displayed_value`。不再使用店小秘采集来的 0 价格、错误变种价、编辑页展示价、`minPrice/maxPrice`、页面缓存价格或采集批次缓存价格。缺少可信 Amazon 页面展示价格或当前任务价格参数时必须阻断保存。
- 物流费：固定写 `0`，因为选品规则要求 FBA/包邮。
- 商家仓库存：固定写任务库存，当前默认 `15`。
- SKU 编码：使用 Amazon ASIN；多 SKU 时按 `ASIN-1`、`ASIN-2` 递增。
- 重量、尺寸：优先使用亚马逊详情页抓到的商品重量和尺寸；缺失时才使用任务默认值。
- 点击“读取 edit.json”后，主插件会自动补当前编辑页可见变种表格，不再需要手动点“诊断补当前页变种”。

### 类目队列

“类目队列”用于正式任务拆分，例如 5-20 美金 730 条、每个类目不超过 15 条：

```text
silicone sink strainer
drawer organizer
desk organizer
cabinet hooks
```

脚本会把它作为补缺口来源。当前类目不够时，不再人工等待或重复采集，而是进入下一个类目继续补。

### 时间目标

当前阶段的时间目标是：

```text
同品类 10 / 20 / 30 条：从 Amazon 点击开始，到进入店小秘速卖通海外托管采集箱，目标 120 秒内。
```

如果超过 120 秒，批次报告和运行指标必须显示卡点，优先优化：

- Amazon 候选链接获取速度
- 店小秘采集开始按钮触发速度
- 采集弹窗完成状态识别
- 批量认领和托管店铺勾选
- 当前类目不足时自动换类目补缺口

## 店小秘确认规则

采集完成后不能只看弹窗成功，必须进入：

```text
产品 > 速卖通海外托管店采集箱
```

确认以下内容：

- 未认领数量是否增加到本批目标数量。
- 列表里是否能看到本批商品。
- 标题、图片、描述、售价是否有内容。
- 如果只进入了部分商品，且这些商品还没有认领成功，保留未认领商品，继续补齐缺口；不要为了凑整批重复删除和重采。

## 批次报告

点击“下载批次报告”会导出当前批次：

- 使用的筛选条件
- 可采集数量
- 待 Logo 复核数量
- 已排除数量
- 已采集数量
- 已认领数量
- 距离目标数量的缺口
- 下一步建议动作
- 每个 ASIN 被排除的原因

这个报告用于判断当前批次是否能进入店小秘采集箱，或者应该换类目继续补量。


## 2026-06-25 正式选品与去重规则

默认任务规则：

- 价格区间默认值：5-20 USD，可按任务配置调整。
- 目标数量默认值：100 个商品，可按任务配置调整。
- 每个品类上限默认值：15 个商品，可按任务配置调整。
- 同一品类内尽量保证商品多样性。

重复识别规则：

- 同一款产品不同颜色，允许作为不同商品分别上架。
- 同一款产品不同颜色，视为不同 SKU，可以分别进入采集流程。
- 同一款产品不同颜色，不属于重复商品，不允许过滤。
- 只在同一个营业执照组内过滤完全相同的 Amazon 链接、完全相同的 ASIN、完全相同的颜色/尺寸/套餐组合、同一商品被重复采集。
- 默认跳过当前营业执照组下已经采集、认领、待发布、已发布或已跳过的完全相同商品。
- 其他营业执照组可以复用相同商品；好卖产品原则上允许跨营业执照组上架，但仍必须通过 Logo/品牌、类目、属性、价格、preflight 和保存闸门。

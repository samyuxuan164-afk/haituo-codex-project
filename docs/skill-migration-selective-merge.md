## 2. 可归档规则

### 2.1 `save.json` 真实成功标准

规则：

HTTP 成功不等于业务成功。  
不能只因为接口返回成功、请求未报错、脚本执行完成，就判断保存或发布成功。  
必须结合以下证据判断：

1. 真实 `save.json` 返回。
2. 页面实际状态。
3. 商品是否进入待发布、发布中、在线、审核中或其他可确认状态。
4. 失败提示、异常状态、字段缺失提示是否已排除。

来源文件：

- `KNOWLEDGE_BASE.md`
- `AGENT.md`
- `codex_skill_pack_v1.md`

合并风险等级：低风险。  
处理建议：可进入归档文件；后续可改写后进入正式规则文件。

### 2.2 `choiceSave.txt` / `choiceSave.zip` / `save.json` 优先级规则

规则：

真实 `save.json` 返回优先于推测、页面感觉、本地构造结果。  
字段和行为来源优先级建议如下：

1. 真实 `save.json` 返回。
2. `choiceSave.txt` / `choiceSave.zip`。
3. `edit.json`。
4. Amazon 商品数据。
5. 类目规则和本地推导规则。

不得反转该优先级。

来源文件：

- `KNOWLEDGE_BASE.md`
- `AGENT.md`

合并风险等级：低风险。  
处理建议：可进入归档文件；后续可进入 `docs/project-execution-rules.md` 或 `docs/schema-current.md`。

### 2.3 不伪造保存成功规则

规则：

不能只因以下情况声明保存成功：

1. 接口有 HTTP 响应。
2. 接口返回格式正常。
3. 脚本没有报错。
4. 页面没有明显报错。
5. 本地报告生成成功。
6. dry-run 构造成功。

保存成功必须有真实业务证据支撑。

来源文件：

- `KNOWLEDGE_BASE.md`
- `AGENT.md`
- `codex_skill_pack_v1.md`

合并风险等级：低风险。

### 2.4 字段结构和值分离规则

规则：

字段结构和字段值必须分开处理。

字段结构可以参考历史样本，包括：

1. 字段所在 JSON 层级。
2. 字段位于 `sku`、`variation`、`optionValues` 或其他结构。
3. 字段在 `choiceSave` 中的位置。
4. 字段在 `save.json` payload 中的位置。

字段值必须按当前商品动态生成，包括：

1. SKU。
2. 价格。
3. 重量。
4. 尺寸。
5. 库存。
6. 运费或物流字段。
7. 变体字段。
8. 当前商品特有属性。

禁止跨商品复用动态业务值。  
禁止硬编码当前商品的 SKU、价格、重量、尺寸、库存等业务值。  
禁止人工补齐未验证的业务字段后声称自动化成功。

来源文件：

- `KNOWLEDGE_BASE.md`
- `AGENT.md`
- `codex_skill_pack_v1.md`

合并风险等级：低风险。  
处理建议：可进入归档文件；后续建议进入 `docs/schema-current.md`。

### 2.5 调试顺序

规则：

遇到保存、字段、payload、类目或发布异常时，建议按以下顺序调试：

1. 先比对真实 `save.json` 返回。
2. 再比对 `choiceSave.txt` / `choiceSave.zip`。
3. 再比对本地生成结果与真实成功样本。
4. 再定位字段结构问题。
5. 最后最小修改插件逻辑。

不得先改主流程，不得跳过真实样本比对。

来源文件：

- `KNOWLEDGE_BASE.md`
- `AGENT.md`

合并风险等级：低风险。  
处理建议：可进入归档文件；后续可进入 `docs/project-execution-rules.md`。

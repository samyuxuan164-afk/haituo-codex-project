## 已确认失败原因

1. 类目没有在编辑页真实选中。
   - 店小秘保存和发布接口不会判断业务类目是否正确。
   - 如果编辑页产品分类为空或错误，保存/发布可能失败，也可能产生错误质量结果。
2. 自定义属性不应默认填写。
   - 本轮失败中自定义属性触发了长度和字符限制。
   - 自定义属性不是核心必填项，默认跳过能降低失败率。
3. 运费模板必须真实选择 `111`。
   - 不能只写入文本，必须触发页面下拉选择并在保存前复核。
4. 必填属性需要保守补齐。
   - 品牌、高关注化学品、产地等字段应使用页面允许值。
5. 保存动作缺少失败后纠偏重试。
   - 遇到未选择、不能为空、字符过长、特殊字符错误时，应先自动修正并重试一次。
6. 模板继承必须验证页面结果。
   - 不能只认为规则已写入或 payload 已构造，必须读页面可见字段确认。

## v1.1.37 修改方案

1. 新增编辑页保存前预检。
   - 检查标题、PC 描述、产品分类、运费模板、自定义属性和页面错误提示。
2. 新增产品分类可见项选择。
   - 根据标题、来源链接、分类文本提取关键词，选择页面可见的最接近类目。
3. 新增必填属性保守补齐。
   - Brand 优先 `None/No Brand`。
   - High-concerned Chemical 优先 `None/天然未处理`。
   - Origin 优先 `Mainland China/中国大陆`。
4. 新增自定义属性清空策略。
   - 默认清空自定义属性输入值，除非后续明确证明平台或类目强制要求。
5. 强化运费模板 `111` 选择验证。
   - 先选择，再读取页面选中状态。
6. 新增保存失败重试。
   - 保存失败后收集页面错误，执行一次预检修复并重试。
7. 新增“预检并保存”入口。
   - 用于明天在编辑页直接验证完整保存链路。

## 预期效果

1. 类目空白导致的保存失败预计明显下降。
2. 自定义属性长度和特殊字符错误预计基本消失。
3. 运费模板误写入但未真实选择的问题预计消失。
4. 单商品编辑页失败后可自动修复一次，减少人工介入。
5. 小批量链路成功率预计从当前不可稳定放大提升到 80%-90% 区间。

## 仍需现场验证的风险

1. 店小秘类目下拉组件可能存在异步加载或虚拟滚动，部分类目仍可能无法自动选中。
2. 不同类目的必填属性名称可能不同，当前只补齐已观察到的高频必填项。
3. 保存失败重试只能处理可见页面错误，隐藏接口错误仍需继续沉淀。
4. 类目业务正确性需要后续接入更强的商品类型到平台类目映射表。

# 店小秘发布自动化规范

本 Skill 用于店小秘待发布、发布失败、在线产品审核状态相关的发布执行与发布前检查。

标准链路：

```text
Amazon 商品 -> 店小秘采集箱 -> 认领 -> 编辑页自动填写 -> 保存 -> 待发布 -> 批次检查 -> 发布
```

## 测试模式

1. 测试阶段允许对 1 个测试商品执行单个发布验证。
2. 单品测试用于验证发布按钮、发布接口、发布结果、发布后状态和错误提示。
3. 单品测试不得扩展为批量发布。
4. 单品测试后必须检查商品最终进入发布中、在线产品、审核中或发布失败。

## 正式批量模式

1. 正式运行阶段默认使用批量发布。
2. 标准操作为：勾选产品 -> 批量操作 -> 批量发布。
3. 正式运行阶段不逐个点击产品级发布。
4. 只有同一批商品全部完成编辑并通过检查后，才允许批量发布。
5. 未完成编辑的商品禁止发布。
6. 如果批次中存在检查失败商品，必须继续编辑或剔除异常商品。

## 批量发布前置条件

批量发布前必须确认：

1. 标题已重写。
2. 商品类型与平台类目匹配。
3. 必填属性完整。
4. PC 描述已重写。
5. 营销图已生成或按规则保留。
6. 运费模板已选择 `111`。
7. 品牌信息已过滤。
8. Logo 产品已过滤。
9. 商品状态正常进入待发布。
10. 特殊字符清洗通过。

## 特殊字符清洗

发布前必须清洗所有即将提交到平台的文本字段，包括标题、PC 描述、属性、规格、图片文字、详情文案和 payload 文本字段。

规则：

1. 禁止英文双引号 `"`。
2. 禁止尺寸符号 `′`、`″`。
3. 禁止中文引号、智能引号。
4. `4.625"`、`4.625″`、`4.625 in.` 必须统一改为 `4.625 inch`。
5. 如果发布前仍检测到上述字符，禁止发布，必须返回编辑页修正后保存。

## 图片与 Brand Story

1. 删除 Brand Story 模块。
2. 保留正常产品图和场景图。
3. 过滤 Logo 图、品牌图、品牌水印图。
4. 选品阶段发现品牌 Logo、水印、品牌名称印刷在产品主体上，直接跳过，不进入采集流程。

## 类目正确性

1. 店小秘和平台发布接口不会自动校验产品类目是否业务正确。
2. 发布成功不代表类目正确。
3. 类目正确性必须由自动化流程在发布前自行校验。
4. 发布前必须检查商品类型与平台类目是否匹配。
5. 如果商品类型与类目明显不一致，禁止发布并记录为类目错误。
6. Sink Strainer 不能归到 Kitchen Faucets。
7. 水槽过滤器不能归到厨房龙头。
8. 类目错误不一定会导致发布失败，但会导致上架质量错误，必须在发布前拦截。

## 自定义属性

1. 编辑页中的自定义属性默认不填写。
2. 自定义属性属于低优先级字段，不作为发布前必填项。
3. 自定义属性经测试容易触发字符长度、特殊字符等校验错误。
4. 自定义属性不影响保存、不影响待发布、不影响正常发布时，统一跳过。
5. 批量运行阶段默认跳过自定义属性，以减少错误率并提升处理效率。

只有以下情况才允许填写自定义属性：

1. 平台强制要求。
2. 类目强制要求。
3. 发布失败明确提示缺少该字段。

## 失败恢复

1. 发布失败后先记录失败原因。
2. 如果失败原因是字段字符校验，返回编辑页清洗字段后保存。
3. 保存后可从发布失败或待发布列表重新执行单品发布测试。
4. 重新发布后必须检查失败原因是否消失，以及是否出现新的平台校验错误。

## 2026-06-24 v1.1.37 发布前拦截规则

1. 正式发布前必须复核编辑页保存结果，不允许把未完成编辑或保存失败的商品带入发布。
2. 批量发布前必须确认同批商品全部通过：类目已选、必填属性完整、标题合规、PC 描述合规、运费模板 111 已选、品牌与 Logo 已过滤、自定义属性为空或有明确强制依据。
3. 如果商品类型与平台类目明显不匹配，禁止发布并记录为类目错误。发布接口成功不代表类目正确。
4. 发布前必须执行特殊字符清洗结果复核，禁止 `"`、`′`、`″`、中文引号和智能引号；尺寸符号统一改为 `inch`。
5. 测试阶段允许单个商品发布验证；正式批量阶段默认使用勾选商品后的批量发布，不逐个点击发布。
## 2026-06-25 发布前标题品牌硬规则

1. 标题禁止保留任何品牌名称、商标、品牌型号。
2. 发布前必须复核标题是否仍包含品牌词、商标词或品牌型号。
3. 如发现品牌词，必须返回编辑页自动重新生成标题并重新保存；未通过前禁止发布。
